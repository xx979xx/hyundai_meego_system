Mad Developer Help

Mad Developer provides developer access to this device and works in stateless mode. When you restart Mad Developer, all configuration information is set to default values. In this case the system state is not changed, the current IP addresses are preserved and DHCP server for usb0 keeps running.

Connecting using USB

To connect your device to your computer using USB:

  1. Plug in USB cable to device and computer.
  2. Close Ovi Suite/Mass Storage popup by tapping outside it.
  3. In Mad Developer window, tap USB Networking.
  4. Tap Windows Network. This is relevant for all supported platforms.
    The network address of the device is displayed on the usb0 line.
  5. Tap Close.
  6. Generate a password for the connection by tapping Developer Password. Use this temporary password to set up SSH RSA/DSA public key authentication.
    Note: Tap Developer Password before configuring an SSH connection on your computer.
  7. Add device IP address and the generated password when configuring device access in your development environment.

Connecting using WLAN

When connecting the device to the computer using WLAN, configuration is automatic when a network is available.

To connect your device to your computer using WLAN:

  1. Make sure the device and the computer are on the same subnet.
    The network address of the device is displayed on the wlan0 line.
  2. Generate a password for the connection by tapping Developer Password. Use this temporary password to set up SSH RSA/DSA public key authentication.
    Note: Tap Developer Password before configuring an SSH connection on your computer.
  3. Add device IP address and the generated password when configuring device access in your development environment.

Note: On the GPRS line, you can check if the device is using the GPRS interface and if the interface is set as default route (R).

Configuration information

The top part the screen shows network configuration information:

  • Interface type (usb0, wlan0, and gprs0)
  • IP address of the device
  • Netmask address of the device
  • Flags
    • S when DHCP server mode is in use
    • D when DHCP client mode is in use
    • R when the default route is set using this interface

Editing USB connection configuration

To edit the USB connection configuration, tap Edit on usb0 line. To change the configuration mode, tap DHCP server:

  • DHCP server is the default mode:
    • DHCP server is in use.
    • Host IP address (192.168.2.15) and netmask (255.255.255.0) are set.
    • Peer IP address (192.168.2.14) is set for DHCP server to use.
  • DHCP client mode: You cannot edit the network configuration settings. Use this mode if your host computer is specially set up as DHCP server for this device.
  • Static mode: Use this mode if:
    • You want to control the interface configuration to, for example, configure a default route.
    • The host computer is not set up as DHCP server for your device but possibly provides routing.

To take your configuration into use, tap Configure. To leave your configuration unchanged, tap Cancel.

Using the device terminal

To use the device terminal, tap Developer Terminal. When the terminal opens, you are logged in as a developer user. This user has the same credentials as normal users, but a different home directory.

Using the device as mass storage

To use the device as mass storage, tap USB Networking > Mass Storage.
